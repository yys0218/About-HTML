About HTML

자바스크립트 JavaScript

HTML: 웹 문서의 내용 구성
CSS: 웹 문서의 레이아웃, 색상, 디자인 구성
Java Script: 웹 문서의 요소에 이벤트 효과(웹의 동작) 담당

- 객체기반의 스크립트 언어
- 타입을 명시할 필요가 없는 인터프리터 언어
>> 브라우저의 자바스크립트 인터프리터가 소스 코드를 바로 실행
- 객체 지향형 프로그래밍(클래스 개념)과 함수형 프로그래밍(function 개념) 모두 표현 가능

자바스크립트의 활용
- 웹 사이트의 UI(User Interface)부분에 많이 사용됨
ㄴ글 정렬방식이 목록형에서 앨범형으로 바뀜/ 버튼을 누르면 아이콘들이 뜸/ 알림창이 뜸 등등
웹사이트 내에서 행해지는 모든 행위
* UI: 메뉴, 텍스트, 팝업창 등 사용자가 사이트를 편리하게 사용할 수 있도록 만드는 모든 디자인적 요소

- 웹 어플리케이션을 만들 때 다양하게 제공되는 API
ㄴ (카카오API, 구글API, 네이버API 등등 오픈소스)

- 활용되는 라이브러리가 다양함
react.js, angular.js, vue.js: 프론트엔드 라이브러리, 프레임워크
node.js :백엔드 서버 개발 프레임워크 - 브라우저 없이도 자바 스크립트를 실행할 수 있는 실행환경(런타임) 제공
ㄴ *클라이언트의 요청을 한 번에 받아 처리 속도가 빠른 것부터 반환함. sns나 채팅서비스에 주로 이용됨
* ~.js : js의 파일확장자

자바스크립트 작성 규칙

- html과 달리, 자바스크립트는 대소문자를 구별함
- 공백과 ; "" 이 안붙어도 작동하는 등, '유연'하기 때문에 오류 찾아내기가 쉽지 않다.
>> 가능한 꼼꼼히 작성할 것!

- 들여쓰기(tab) 해주기
- ; 으로 실행문 구분
- 적절하게 공백이 사용되어야함. var name = "값";
>> 예약어 val name, 연산자 = ,값 사이에 사용하여 가독성 있게 작성
